# 中文优先（zh-prefer）

非必要不用英文。对人读的自然语言默认中文；机器约定保持英文。

## 规则

| 类别 | 语言 |
|------|------|
| 对话回复 | 中文 |
| 文档（README、规格、说明书、设计稿说明等） | 中文 |
| 代码注释 | 中文 |
| UI 文案、错误提示、可读日志信息 | 中文 |
| 函数/变量/类/文件名等标识符 | 英文 |
| HTTP/JSON 字段、环境变量名等协议与配置键 | 英文 |
| commit message、PR 标题与正文 | 跟随仓库既有约定；无约定则用中文 |

专有名词、库名、API 名、错误码、需原文引用的术语可保留英文，不强行翻译。

## 旧内容

只约束**本次新写**或用户**明确要求改写**的自然语言。不要顺手把整文件/整仓英文翻译成中文。

## 覆盖

用户点名某产物用英文时（如「README 用英文」），仅该产物改用英文；未点名的对话与其它文件仍按中文优先。

仅当用户明确说「本轮全英文」「后面都用英文」时，才对整段会话放宽。

## 自检

写出或改完自然语言前快速确认：

- [ ] 注释/文档/对话是否无故英文化？
- [ ] 是否误把标识符或协议字段改成了中文？
- [ ] 是否擅自翻译了未要求改写的旧英文内容？
